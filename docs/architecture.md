## DynamoDB Schema

### Tasks Table
- **Partition Key**: `userId` (String)
- **Sort Key**: `id` (String)
- **GSI - StatusIndex**: userId + status
- **GSI - DueDateIndex**: userId + dueDate
- **Attributes**: title, description, priority, status, category, tags, dueDate, estimatedMinutes, progress, subtasks, order, createdAt, updatedAt, completedAt

### Plans Table
- **Partition Key**: `userId` (String)
- **Sort Key**: `date` (String, YYYY-MM-DD)
- **Attributes**: timeBlocks[], availableHours, energyPattern, createdAt

### Insights Table
- **Partition Key**: `userId` (String)
- **Sort Key**: `createdAt` (String, ISO timestamp)
- **TTL**: `ttl` (auto-expire after 30 days)
- **Attributes**: type, title, content, confidence, metadata

### Notifications Table
- **Partition Key**: `userId` (String)
- **Sort Key**: `id` (String)
- **TTL**: `ttl` (auto-expire after 7 days)
- **Attributes**: type, title, message, read, taskId, createdAt

## AI Agent Architecture

Each agent uses structured prompts with:
1. **System prompt** - Agent role, rules, response format
2. **Context block** - Current tasks, user preferences, time
3. **User message** - The specific request

All responses are JSON-only with confidence scores. Chain-of-thought reasoning is used internally but never exposed.

## Security Model

1. **Authentication**: Cognito issues JWTs after email/password auth
2. **Authorization**: API Gateway validates JWT before Lambda execution
3. **Data isolation**: DynamoDB uses userId as partition key - users can only access their own data
4. **Input validation**: Zod schemas validate all request bodies
5. **Least privilege**: Each Lambda has only the IAM permissions it needs
6. **Transport**: HTTPS enforced, HSTS headers set

## Performance Considerations

- **Cold starts**: ARM64 + small bundle size minimizes cold start (~200ms)
- **DynamoDB**: Single-digit millisecond latency at any scale
- **Bedrock**: Nova Lite has fast inference (~1-3s for typical requests)
- **Frontend**: Vite code splitting, React Query caching, optimistic updates
- **Connection reuse**: AWS SDK connection pooling enabled
